# 控制面板模块适用性与中文日志设计

## 目标

- 面板业务日志、失败明细和无数据明细使用任务中文名，不展示任务 ID。
- 账号选定后，禁用并取消勾选不属于该账号来源的模块。
- 周期切换后，禁用并取消勾选不支持该周期的模块。
- “全选模块”只选择当前可用模块。

## 数据来源

不增加配置。继续使用 `config/tasks.json` 已有字段：

- `task_name`：模块中文名。
- `account_source`：模块适用的账号池。
- `frequency`：模块支持的周期。

账号名称可能同时存在于多个账号池，面板按其全部来源判断模块适用性。

## 页面行为

平台变化时，账号列表展示该平台全部模块涉及的账号并保持去重，不再由当前勾选模块反向裁剪。

账号或周期变化时重新计算模块状态：只要至少一个已选账号属于模块的 `account_source`，且当前周期在模块 `frequency` 中，该模块即可选择；否则取消勾选并禁用。未选择账号或模块时不允许启动，避免空筛选被底层解释成运行全部账号或全部模块。

## 后端校验

启动手动任务前，后端使用同一份任务和账号配置校验所选模块、账号及周期。非法组合返回业务可读错误，不启动 `main.py`。该校验用于防止绕过页面或页面状态不同步。

## 中文日志

从当前环境任务配置构造 `任务 ID -> task_name` 映射。面板生成运行摘要和业务日志时替换最终失败、无数据明细中的任务 ID；磁盘中的原始运行日志保持不变。

## 测试

- 任务 ID 在摘要和业务日志中转换为中文名。
- 多账号源重复账号能匹配其全部适用模块。
- 不属于所选账号或不支持所选周期的模块被判定为不可用。
- 非法组合在启动子进程前被拒绝。
- 页面包含账号/周期变化时刷新模块状态以及“全选仅选择可用项”的行为。
